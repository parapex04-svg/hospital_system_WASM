# 🏥 Hospital Patient Registration & Follow-up Tracker

> A full-featured console-based C program compiled to **WebAssembly** and deployed via **GitHub Pages** — running entirely in the browser with no server required.

---

## 📸 Overview

This project implements a hospital patient management system written in **C (C99)**, compiled to WebAssembly using **Emscripten**, and wrapped in a custom-designed **phosphor-terminal UI** styled to look like a 1980s medical mainframe.

The C program runs live in your browser. All patient data, visit records, and file I/O happen inside the Emscripten virtual filesystem (MEMFS) — no backend, no database, no cloud.

---

## ✨ Features

| # | Feature | Description |
|---|---------|-------------|
| 1 | **Register Patient** | Add patients with ID, name, age, and phone number |
| 2 | **Add Visit Record** | Log date, diagnosis, and prescription per patient |
| 3 | **Search Patient** | Find any patient instantly by their unique ID |
| 4 | **Display All Patients** | Tabular view of every registered record |
| 5 | **Visit History** | Full chronological visit log per patient |
| 6 | **Frequent Visitors** | Auto-flags patients with more than 3 visits |
| 7 | **Save to File** | Persist data to `patients.txt` + `visits.txt` in WASM FS |
| 8 | **Load from File** | Restore saved records across the session |
| 9 | **Exit** | Auto-saves before shutting down the C runtime |

## 🏗️ Building from Source

### Prerequisites

- [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html) installed and activated
- `emcc` available on your `PATH`

```bash
# Verify Emscripten is ready
emcc --version
```

### Option A — Custom shell (recommended)

Keeps your styled `index.html` and generates only the JS/WASM loader:

```bash
emcc main.c \
  --shell-file index.html \
  -o index.js \
  -s EXPORTED_RUNTIME_METHODS='["FS"]' \
  -s ALLOW_MEMORY_GROWTH=1
```

This produces `index.js` + `index.wasm`. Open `index.html` in a browser (served via a local server — see below).

### Option B — Quick single-command build

Let Emscripten generate its own HTML (no custom shell):

```bash
emcc main.c -o hospital.html
```

Then open `hospital.html` directly. You can replace the generated HTML with the custom `index.html` afterwards.

### Option C — Native build (for local testing without WASM)

```bash
gcc -Wall -Wextra -std=c99 -o hospital main.c
./hospital
```

Useful for rapid input/output testing before compiling to WASM.

## 🌐 Running Locally

WebAssembly requires files to be served over HTTP — opening `index.html` directly as a `file://` URL will not work due to browser CORS restrictions.

**Python (quickest):**
```bash
python3 -m http.server 8080
# then open http://localhost:8080
```

**Node.js:**
```bash
npx serve .
```

**VS Code:** Install the [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer) extension and click **Go Live**.

## 🚀 Deploying to GitHub Pages

1. Build the project (Option A or B above)
2. Make sure your repo contains: `index.html`, `index.js`, `index.wasm`
3. Push to GitHub
4. Go to **Settings → Pages → Source** → set branch to `main`, folder to `/ (root)`
5. GitHub Pages will serve your site at `https://<username>.github.io/<repo>/`

```bash
git add index.html index.js index.wasm main.c README.md
git commit -m "Deploy hospital tracker WASM"
git push origin main
```

## 🧱 Technical Design

### Structs

```c
struct Patient {
    int  id;
    char name[50];
    int  age;
    char phone[15];
};

struct Visit {
    int  patientId;
    char date[20];
    char diagnosis[100];
    char prescription[100];
};
```

### Architecture

```
Browser
  └── index.html  (phosphor-terminal UI shell)
        ├── Module.print()       ← captures C stdout → textarea#output
        ├── Module.printErr()    ← captures C stderr → textarea#output
        └── index.js / index.wasm
              └── main.c (C runtime)
                    ├── patients[]   in-memory array (max 200)
                    ├── visits[]     in-memory array (max 1000)
                    └── MEMFS        virtual file system
                          ├── patients.txt
                          └── visits.txt
```

### Input Validation

| Field | Rule |
|-------|------|
| Age | Must be between 1 and 120 |
| Phone | Exactly 10 numeric digits |
| Patient ID | Positive integer, must be unique |
| Menu choice | Must be 1–9, invalid choices re-prompt |

### File Format

```
patients.txt  →  101,Alice Johnson,34,9876543210
visits.txt    →  101,2026-03-20,Fever,Paracetamol 500mg
```

### Functions Implemented

```
addPatient()          addVisit()            searchPatient()
displayPatients()     displayVisits()       countVisits()
showFrequentVisitors() savePatientsToFile() saveVisitsToFile()
loadPatientsFromFile() loadVisitsFromFile() validateAge()
validatePhone()        isUniquePatientID()  flushInputBuffer()
```

---

## ⚠️ Limitations

- **MEMFS is session-only.** Files saved via menu options 7/8 persist within the browser tab's session but are lost on page refresh. To persist across reloads, the build would need Emscripten's IDBFS (IndexedDB-backed filesystem).
- **No concurrent users.** This is a single-user local runtime — the C program runs as a single instance per browser tab.
- **Max records:** 200 patients, 1000 visits (configurable via `#define` in `main.c`).
